Requirements


Are you ready to submit?

Before you do, take a look at what we expect from submissions!

If you're submitting for a workshop...


For this, you must have a frontend with:

  • a form which contains at least 3 inputs and a submit button
  • an image
  • at least one sentence of text
  • JS that adds something to the website based on the form contents when its submitted
  • At least 4 CSS rules

Have a backend which:

  • "serves" your frontend (or in simpler words, allows people to access your frontend)
  • two API endpoints implementing creating and reading of something

Have a basic full stack app where, at minimum,

  • the frontend communicates to the backend for functionality using API endpoints
  • said communication provides functionality for the frontend
  • it is sufficiently customized!

Make the website look unique (make it your own!) and add at least one extra API endpoint on the backend

If you're submitting individually...

You don't need to submit for these! You'll get all your rewards at the end instead :D
However, you still will need, at minimum, two API endpoints (one to create and one to read something) on a Python backend which should also serve your website, along with a frontend with at least 3 form inputs, a submit button, some text, and an image, along with JS to provide functionality.

Have a basic full stack app where, at minimum

  • the frontend communicates to the backend for functionality using API endpoints
  • said communication provides functionality for the frontend
  • you haven't copied from other people! (use common sense for this rule)
